🚀 CollabSpace

Sistema de Colaboración en Tiempo Real

Proyecto Académico - Interacción Hombre-Máquina
⚡ Enfoque MVP - Producto Mínimo Viable
👥 Equipo de 5 Desarrolladores | ⏱️ Duración: 4 semanas (1 mes)

📋 Resumen Ejecutivo

CollabSpace es una plataforma web de colaboración en tiempo real que permite a equipos trabajar simultáneamente en un whiteboard digital y comunicarse mediante chat instantáneo. El proyecto adopta un enfoque de Producto Mínimo Viable (MVP) para asegurar la entrega de funcionalidades core bien implementadas dentro del plazo de 4 semanas.

🎯 Filosofía MVP del Proyecto

Ante la restricción temporal de 1 mes, el equipo ha decidido enfocarse en implementar pocas funcionalidades pero excelentes en lugar de muchas features mediocres. El objetivo es entregar un whiteboard colaborativo completamente funcional con chat en tiempo real, demostrando dominio técnico en WebSockets y proporcionando una experiencia de usuario pulida.

🎓 Contexto Académico

Proyecto final para la materia de Interacción Hombre-Máquina. Énfasis en diseño de interfaces intuitivas, experiencia de usuario fluida, comunicación en tiempo real mediante WebSockets, y trabajo colaborativo en equipo de 5 desarrolladores con metodología ágil.

🎯 Objetivos del Proyecto

  • Objetivo Principal: Desarrollar una plataforma web MVP funcional que demuestre colaboración en tiempo real con un whiteboard digital y chat instantáneo.
  • Objetivo Técnico: Implementar comunicación bidireccional mediante WebSockets (Flask-SocketIO) con sincronización en tiempo real de menos de 500ms de latencia.
  • Objetivo de UX/UI: Crear una interfaz intuitiva y visualmente atractiva que cualquier usuario pueda usar sin capacitación previa.
  • Objetivo Académico: Demostrar comprensión de principios de interacción hombre-máquina, arquitectura cliente-servidor, y desarrollo full-stack moderno.
  • Objetivo Realista: Entregar un proyecto completo y funcional en 4 semanas, evitando el "feature creep" y priorizando calidad sobre cantidad.

Funcionalidades Detalladas

🎨 Whiteboard Colaborativo

Canvas digital con Fabric.js donde múltiples usuarios dibujan simultáneamente. Herramientas: lápiz libre, selector de colores, grosor de línea, borrador. Sincronización en tiempo real mediante WebSocket con throttling optimizado.

💬 Chat en Tiempo Real

Sistema de mensajería instantánea en sidebar. Mensajes persistentes en base de datos. Notificaciones visuales de nuevos mensajes. Scroll automático al último mensaje.

🚪 Gestión de Salas

Crear salas con nombre personalizado y código único de 6 dígitos. Unirse a salas mediante código. Sistema simple sin roles complejos (todos son participantes).

👥 Presencia de Usuarios

Lista en vivo de usuarios conectados en la sala. Cada usuario tiene un color único. Indicadores de conexión/desconexión en tiempo real.

🔐 Autenticación Simple

Registro e inicio de sesión básico con JWT. Hashing de contraseñas con Werkzeug. Sesión persistente en localStorage del navegador.

🖱️ Cursores Colaborativos

Ver el cursor de cada usuario en el canvas en tiempo real. Cada cursor tiene el color único del usuario y muestra su nombre.

📐 Formas Geométricas

Herramientas adicionales para dibujar círculos, rectángulos y líneas rectas. Solo si sobra tiempo en Semana 4.

📝 Notas Compartidas

Editor de texto simple con Quill.js para tomar notas colaborativas. Feature de bonus si el proyecto va adelantado.

📅 Cronograma Realista (4 Semanas)

Semana 1: Fundamentos

Días 1-7 | Setup & Base

Backend: Flask configurado, PostgreSQL conectado, modelos (User, Room, Message), API REST básica (auth, rooms).

Frontend: React con Vite, React Router, páginas base (login, home, room), Context API setup.

Whiteboard: Fabric.js funcionando, canvas dibuja localmente, UI básica de herramientas.

UI/UX: Wireframes, mockups en Figma, Tailwind configurado, componentes base (Button, Input, Card).

QA: Git configurado, README inicial, plan de testing.

✅ Milestone: Autenticación funciona, se puede crear/unir salas (sin WebSocket aún).

Semana 2: Tiempo Real

Días 8-14 | WebSocket Magic

Backend: Flask-SocketIO configurado, eventos (connect, disconnect, join_room, send_message, draw).

Frontend: Socket.io-client integrado, conexión a WebSocket, manejo de eventos.

Whiteboard: Eventos del canvas enviados por WebSocket, recepción y renderizado de dibujos de otros.

UI/UX: Página principal de sala completa, chat sidebar, lista de usuarios online.

QA: Testing de WebSocket con múltiples usuarios, encontrar bugs críticos.

✅ Milestone: Chat funciona en tiempo real, whiteboard sincroniza entre 2 usuarios.

Semana 3: Features Core

Días 15-21 | Polish & Optimize

Backend: Optimización de sincronización, throttling de eventos, persistencia de mensajes.

Frontend: Todas las herramientas del whiteboard funcionando, colores, grosor, borrador.

Whiteboard: Cursores de otros usuarios visibles, smooth drawing, optimización de performance.

UI/UX: Responsive design completo, animaciones sutiles, pulir detalles visuales.

QA: Testing exhaustivo con 5 usuarios simultáneos, mobile testing, bugs críticos resueltos.

✅ Milestone: Aplicación completamente funcional, lista para demo (falta solo deployment y documentación).

Semana 4: Deployment & Presentación

Días 22-28 | Ship It!

Backend: Deploy en Render.com, configurar PostgreSQL cloud, variables de entorno.

Frontend: Build de producción, deploy en Vercel, conectar con backend en producción.

Whiteboard: Testing final, agregar formas geométricas SI sobra tiempo.

UI/UX: Últimos ajustes visuales, dark mode SI sobra tiempo, loading states.

QA: Documentación completa, video demo grabado y editado, presentación preparada (10 slides).

✅ Entregable Final: App desplegada en la nube, video demo, presentación lista, repositorio documentado.
